# Sales Data Quality Lab

This lab asks students to build an Apache Airflow DAG that validates incoming sales order data, writes a JSON summary, and sends a Discord notification for both pass and fail outcomes.

Status as of 2026-05-14:
- Lab package: ready
- Reference implementation: verified locally and in Airflow
- Real Discord delivery: depends on a valid webhook URL

## Learning Outcomes

- Build a simple Airflow DAG with Python tasks.
- Read CSV input data and enforce basic quality rules.
- Generate a machine-readable validation summary in JSON.
- Stop the pipeline when bad data is detected.
- Send operational alerts to Discord with a webhook.

## Deliverables

- `sales_data_quality_pipeline.py`
- `src/config.py`
- `src/validation.py`
- `orders_failed.csv`
- `orders_passed.csv`
- `validation_summary.json` generated by the DAG
- Discord alerts for success and failure runs

## Validation Rules

1. `customer_id` must not be empty.
2. `amount` must be numeric and greater than `0`.
3. `status` must be one of `completed`, `pending`, or `cancelled`.
